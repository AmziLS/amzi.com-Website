ARulesXL turns Microsoft Excel into a powerful tool for delivering business rule applications that combine decision support with computational analysis.

Instead of difficult to code, maintain and understand Excel IF statements, rules are entered directly on your spreadsheet in a natural and easy to understand syntax. The rules can then be queried, much like a database. (Click on the image below to see a simple example contrasting different approaches.)

Rules entered using ARulesXL are:

  • Easier to code,
  • Easier to maintain,
  • More reliable, and
  • Easily audited.

View simple examples illustrating ARulesXL for loan approval or call pricing.

ARulesXL is ideal for Sarbanes Oxley compliant spreadsheets with easy to read and audit business rules that generate or verify results. See an example.

See our whitepaper on Logical and Procedural Knowledge: Business Rules and Excel.

Rule sets that are created in Excel can be exported to run in a variety of tools and environments.

  • In Excel — Rules can be run in Excel, and spreadsheet users can change the inputs to see different results. The rules can be optionally encoded and hidden, preventing Excel users from reading or modifying them.

  • On the Web — Rule sets can be called from Web applications written in .NET or Java.

  • In Any Application — Rule sets can be used in Extend models where they are most useful for simulating human decision making. They can also be queried from any application written in C++, C#, Delphi, Java, VB and more.

If you want to run your rule sets in a tool not listed here, please contact us.
