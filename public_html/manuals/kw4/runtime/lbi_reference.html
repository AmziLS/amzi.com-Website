LBI Reference

Preparing to Run a Knowledgebase

Before the knowledgebase can be run, the LBI must be initialized, a knowledgebase file opened and session started. The following LBI calls provide those services.

initialize

lbi(0, initialize([directory = <pathname>, log_file = <log_file>, message_level = <level>]), INFO)

Parameters:

  • <pathname> The fully qualified name of the directory where the knowledgebase is located and the log file will be opened.
  • <log_file> The name of the log file.
  • <level> The message level for the log file: hi, medium, low, none.

Returns the property list INFO containing:

  • system = <string> The name of the reasoning engine.
  • version = <string> The version number of the reasoning engine.

Initializes the reasoning engine.

open

lbi(0, open(<knowledgebase_file>), _)

Parameters:

  • <knowledgebase_file> The name of the knowledgebase file.

Returns:

  • Nothing.

Opens the specified knowledgebase file.

new_session

lbi(<id>, new_session, _)

Parameters:

  • <id> The identifier for this session.

Returns:

  • Nothing.

Creates a new reasoning session with the specified identifier. The identifier is used in all subsequent LBI calls.

Running the Knowledgebase

solve

lbi(<id>, solve, MORE)

Parameters:

  • <id> The identifier for this session.

Returns:

  • MORE = more if the reasoning process has more work to do.

Tells the inference engine to start or continue reasoning. After each call to solve, get_action needs to be called and acted upon until there are no more actions.

Interacting with the User

get_action

lbi(<id>, get_action, ACTION)

Parameters:

  • <id> The identifier for this session.

Returns the property list ACTION containing:

  • ask(user, <property_list>) The list of properties from the question object that the reasoning engine needs a value for.
  • tell(user, <property_list>) The list of properties from the text object for the goal the reasoning engine has deduced is true.
  • none No more actions to perform. Call solve again.

Returns an action to be performed. It can be a question to ask the user, in which case the reasoning engine expects an LBI assert to tell the reasoning engine what the user answered. The action can also be a conclusion the reasoning engine has determined is true. If that conclusion is associated with a text object, the fields of that object are returned.

Asserting New Fact Values

lbi(<id>, assert(fact(<name>, <value>)), _)

Parameters:

  • <id> The identifier for this session.
  • <name> The name of the question (fact).
  • <value> The value selected by the user using appropriate quoting, if needed, for strings, and square brackets for lists (multiple values).

Returns:

  • Nothing.

Asserts a new fact value for the reasoning engine to use. String values should be passed in $'s. Lists of values are passed in square brackets, separated by commas (also quoted if necessary).
