  Dynamic Database

Running Prolog

Source files are consulted into an in-memory database called the dynamic database, or logicbase.

The consulted clauses are called dynamic clauses because they can be dynamically created and/or deleted.

The Listener is an interactive environment for querying and modifying the dynamic database.

The Debugger is an extension of the listener for tracing Prolog execution.

The Compiler compiles predicates into byte codes that can be loaded into the dynamic database. Compiled predicates are called static because their clauses cannot be modified.

An application can contain both static and dynamic predicates.

The details of how the compiler works is not ISO standard and might be different for SICStus.

Typically, the interpreter is used for development (and this class); the compiler for deployment.

Eclipse allows projects composed of multiple files to be consulted or compiled together.
