Applying Logical Knowledge

  Database Approach for Logical Knowledge

Logical relationships can be expressed as a data table:

Weekend Night Price
true true 5
true false 5
false true 7
false false 10

Limitations

  • Gets difficult as number of parameters increase
  • Further complicated if only subsets of parameters used for some rules
  • Note need for two rows to capture the single weekend price rule
  • The clarity of original specification starts to get lost in the table
