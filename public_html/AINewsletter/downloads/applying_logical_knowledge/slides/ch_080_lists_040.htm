Applying Logical Knowledge

  Lists

Lists are nested structures, just like Turkish dolls, but with a syntactic sugar that makes them more readable. They are written like:

[apple, pear, plum, cherry]

But can really be thought of as:

c( apple, c( pear, c( plum, c( cherry, c))))

A special syntax gets the first and second arguments:

[HEAD | TAIL]

HEAD - the first element of the list.
TAIL - the LIST of remaining elements.

Which is the same as:

c(HEAD, TAIL)

?- [HEAD|TAIL] = [apple, pear, plum, cherry].
HEAD = apple
TAIL = [pear, plum, cherry]

?- c(HEAD,TAIL) = c( apple, c( pear, c( plum, c( cherry, c)))).
HEAD = apple
TAIL = c( pear, c( plum, c( cherry, c)))

A special token [] (which is an atom) is the empty list, same as c the wooden doll.

The TAIL of a list is ALWAYS another list, or the empty list, so it can be fed to recursive predicates.
