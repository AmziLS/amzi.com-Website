Applying Logical Knowledge

  Arithmetic Operators

By themselves mean nothing, just symbols.

Only "evaluated" in the context of arithmetic built-in predicates:

  • is
  • <  >  >=  =<  =:=
?- X = 3+4.
X = 3+4.
   
?- X is 3+4.
X = 7.
   
?- 3 + 4 =:= 4 + 3.
yes

?- 3 + X =:= 3 + 4.
system_error
506
Invalid numeric value in evaluable term: evaluating expression in term: 3 + H118
While executing: =:=/2

?- 3 + X = 3 + 4.
X = 4 
yes

?- apple + X = apple + orange.
X = orange 
yes

You can use them for readability, so rather than:

city(lax, 5, 13).

You can write

city(lax, 5:13).

Also good for writing multiple terms in a single write/1 goal:

write(TO:FROM - CARRIER:FLIGHT_NUMBER).
