Applying Logical Knowledge

  Procedural Approach

The rules can be coded as a decision tree using procedural if-then-else statements:

if day = weekend
   then price = 5
else
   if time = night
      then price = 7
      else price = 10

Or should it be:

if time = night
   if day = weekend
      then price = 5
      else price = 7
else
   if day = weekend
      then price = 7
	  else price = 10

Limitations

  • If-then-else has control flow branching semantics, not pattern-matching
  • Require arbitrary decisions on nesting attributes to be tested
  • Original rules difficult to "see" in the code
  • Hard to get right (find the bug above)
  • Hard to maintain
