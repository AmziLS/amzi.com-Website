Applying Logical Knowledge

  Built-In Predicates

Built-in predicates are part of the Prolog environment. They can be queried like normal predicates and provide various services. Many are used to create "side effects", such as input/output, and do not affect logical inferencing.

There is an ISO standard set of built-in predicates but most Prolog implementations have additional, non-standard, ones as well.

Here's some common ones:

  • write(T) writes a term to the console
  • nl outputs a new line
  • tab(N) tabs N spaces
  • X = Y unifies X and Y
  • true always succeeds
  • fail always fails
  • not X logical NOT, reverse true/fail
  • repeat always succeeds when backtracked into

Try these queries in the listener:

  • ?- write('Hello World').
  • ?- write('Hello'), nl, write('World').
  • ?- write( astructure(Var1, Var2) ).
  • ?- true.
  • ?- fail.
  • ?- not fail.
  • ?- X = hello.
  • ?- this(that) = Y.
  • ?- this = that.
  • ?- not this = that.
